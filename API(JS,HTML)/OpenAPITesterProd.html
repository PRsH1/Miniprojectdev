eformsign API Tester — 사용 가이드

                
7단계로 빠르게 API를 테스트해 보세요

            

            
        

        

            


                
                

                    
1

                    
                    
환경 선택

                    
화면 우측 상단에서 호출할 서버 환경을 선택합니다.

                    
    
                        
  • 운영 (SaaS) — 일반 eformsign 환경
    • 
                        
  • 공공 (CSAP) — 공공기관 전용 환경
    • 
                        
  • 직접 입력 — 커스텀 도메인 지정
    • 
                    

                


                
                

                    
2

                    
                    
인증 설정 및 토큰 발급

                    
상단 인증 패널을 열어 정보를 입력하고 Access Token을 발급받습니다.

                    
    
                        
  • Signature 방식 — API Key + User ID + 비밀 키(Hex) 입력
    • 
                        
  • Bearer 방식 — 기발급된 Access Token 직접 입력
    • 
                        
  • 발급 후 토큰은 자동으로 Authorization 헤더에 삽입됩니다
    • 
                        
  • 토큰 표시 영역을 클릭하면 클립보드에 복사됩니다
    • 
                    

                


                
                

                    
3

                    
                    
인증 정보 저장 / 불러오기

                    
자주 사용하는 인증 정보를 이름 붙여 저장해두면 다음에 빠르게 불러올 수 있습니다. (로그인 필요)

                    
    
                        
  • 인증 저장 버튼으로 현재 입력된 API Key, User ID, 비밀 키를 이름과 함께 서버에 저장합니다.
    • 
                        
  • 인증 불러오기 버튼으로 목록에서 선택하면 인증 패널 필드가 자동으로 채워집니다.
    • 
                        
  • 비밀 키 저장 여부를 선택할 수 있으며, 미저장 시 불러오기 후 직접 입력이 필요합니다.
    • 
                        
  • 로그인한 계정에 귀속되며 다른 기기에서도 불러오기가 가능합니다.
    • 
                    

                


                
                

                    
4

                    
                    
API 선택

                    
왼쪽 사이드바에서 호출할 API를 선택합니다.

                    
    
                        
  • 그룹, OPA 코드, HTTP Method 별로 묶여 있으며 클릭하여 접고 펼칠 수 있습니다
    • 
                        
  • 상단 검색창에서 API 이름 또는 OPA 코드로 검색 가능합니다
    • 
                        
  • 사이드바 너비는 경계선을 드래그해 조절할 수 있습니다
    • 
                        
  • 요청 영역과 응답 영역 사이의 경계선을 드래그하면 각 영역의 높이를 조절할 수 있습니다
    • 
                    

                


                
                

                    
5

                    
                    
요청 구성

                    
Path / Query / Headers / Body 탭에서 요청 내용을 설정합니다.

                    
    
                        
  • Path — URL 경로에 포함되는 파라미터 입력 (예: {document_id})
    • 
                        
  • Query — URL 뒤에 붙는 Query String 입력
    • 
                        
  • Headers — 요청 헤더 추가 및 수정
    • 
                        
  • Body — JSON 요청 본문 입력 (정렬/초기화 지원)
    • 
                        
  • 예시 응답 버튼으로 성공/실패 응답 구조 미리 확인 가능
    • 
                    

                


                
                

                    
6

                    
                    
요청 전송 및 응답 확인

                    
Send 버튼을 눌러 API를 호출하고 응답을 확인합니다.

                    
    
                        
  • 상태 코드 · 응답 시간 · 응답 크기가 표시됩니다
    • 
                        
  • 응답이 PDF/ZIP 파일이면 자동으로 다운로드됩니다
    • 
                        
  • Send and Download — 응답을 파일로 저장
    • 
                        
  • Code 버튼으로 현재 요청의 코드 스니펫을 확인할 수 있습니다 (cURL · JS fetch · JS jQuery · Python · Java 5개 언어 지원)
    • 
                        
  • API 명세 버튼으로 요청/응답 명세를 모달로 확인할 수 있습니다
    • 
                    

                


                
                

                    
7

                    
                    
요청 저장 및 히스토리

                    
자주 쓰는 요청을 저장하고 언제든 불러올 수 있습니다.

                    
    
                        
  • Save 버튼으로 현재 환경·파라미터·헤더·Body를 통째로 저장
    • 
                        
  • 저장 시 이름을 직접 입력하거나 기본값(API 이름)을 그대로 사용 가능
    • 
                        
  • 저장된 API 항목에 카운트 배지가 생기고, 클릭하면 저장 목록이 펼쳐짐
    • 
                        
  • 사이드바 저장됨 탭에서 전체 이력을 최신순으로 확인, 개별·전체 삭제 지원
    • 
                        
  • localStorage에 저장되므로 새로고침 후에도 유지됩니다 (최대 100개)
    • 
                    

                


            


            

                
                

                    Tip.
                    OPA2_007 (새 문서 작성 - 외부), OPA2_048 (외부자 반려)는 Access Token 대신 Company API Key를 Base64 인코딩하여 Bearer 토큰으로 사용합니다.
                    인증 패널에서 토큰을 발급하지 않고, Headers 탭의 Authorization 값을 직접 수정하세요.
