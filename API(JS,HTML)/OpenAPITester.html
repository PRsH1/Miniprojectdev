eformsign API Tester — 사용 가이드

                
5단계로 빠르게 API를 테스트해 보세요

            

            
        

        

            


                
                

                    
1

                    
                    
환경 선택

                    
화면 우측 상단에서 호출할 서버 환경을 선택합니다.

                    
    
                        
  • 운영 (SaaS) — 일반 eformsign 환경
    • 
                        
  • 공공 (CSAP) — 공공기관 전용 환경
    • 
                        
  • 직접 입력 — 커스텀 도메인 지정
    • 
                    

                


                
                

                    
2

                    
                    
인증 설정 및 토큰 발급

                    
상단 인증 패널을 열어 정보를 입력하고 Access Token을 발급받습니다.

                    
    
                        
  • Signature 방식 — API Key + User ID + 비밀 키(Hex) 입력
    • 
                        
  • Bearer 방식 — 기발급된 Access Token 직접 입력
    • 
                        
  • 발급 후 토큰은 자동으로 Authorization 헤더에 삽입됩니다
    • 
                    

                


                
                

                    
3

                    
                    
API 선택

                    
왼쪽 사이드바에서 호출할 API를 선택합니다.

                    
    
                        
  • 그룹별로 묶여 있으며 클릭하여 펼칠 수 있습니다
    • 
                        
  • 상단 검색창에서 API 이름 또는 OPA 코드로 검색 가능합니다
    • 
                        
  • 사이드바 너비는 경계선을 드래그해 조절할 수 있습니다
    • 
                    

                


                
                

                    
4

                    
                    
요청 구성

                    
Path / Query / Headers / Body 탭에서 요청 내용을 설정합니다.

                    
    
                        
  • Path — URL 경로에 포함되는 파라미터 입력 (예: {document_id})
    • 
                        
  • Query — URL 뒤에 붙는 Query String 입력
    • 
                        
  • Headers — 요청 헤더 추가 및 수정
    • 
                        
  • Body — JSON 요청 본문 입력 (정렬/초기화 지원)
    • 
                        
  • 예시 응답 버튼으로 성공/실패 응답 구조 미리 확인 가능
    • 
                    

                


                
                

                    
5

                    
                    
요청 전송 및 응답 확인

                    
Send 버튼을 눌러 API를 호출하고 응답을 확인합니다.

                    
    
                        
  • 상태 코드 · 응답 시간 · 응답 크기가 표시됩니다
    • 
                        
  • 응답이 PDF/ZIP 파일이면 자동으로 다운로드됩니다
    • 
                        
  • Send and Download — 응답을 파일로 저장
    • 
                        
  • Code 버튼으로 현재 요청의 코드 스니펫을 확인할 수 있습니다
    • 
                    

                


            


            

                
                

                    Tip.
                    OPA2_007 (새 문서 작성 - 외부)는 Access Token 대신 Company API Key를 Base64 인코딩하여 Bearer 토큰으로 사용합니다.
                    인증 패널에서 토큰을 발급하지 않고, Headers 탭의 Authorization 값을 직접 수정하세요.
